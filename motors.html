Motors

As I said earlier, this is based on the hardware from Rev Robotics, and will be using stuff as such.

SparkMax

The SparkMax is a motor controller from Rev Robotics which connects to a motor. The SparkMax object is constructed with an id, which must be set up in the Rev Hardware Client. The import for the SparkMax class, which should be placed with the other imports in the Robot.java file, is as follows: 

            
import com.revrobotics.spark.SparkMax;
import com.revrobotics.spark.SparkLowLevel.MotorType;

Once you have the import placed in your file, you are ready to start creating motors. To create a SparkMax object, we need to create the object in the class and then initialize the object inside of the constructor. Your Robot.java file should look as follows (with some methods also): 

            
package frc.robot;

import com.revrobotics.spark.SparkMax;
import com.revrobotics.spark.SparkLowLevel.MotorType;

import edu.wpi.first.wpilibj.TimedRobot;

public class Robot extends TimedRobot {
  private SparkMax motor;

  public Robot() {
    motor = new SparkMax(0,MotorType.kBrushless); 
    //0 is the ID of the SparkMax motor controller, set in the Rev Hardware Client
    //MotorType.kBrushless is the type of motor it is,
    //which can usually be found on the page where the motor was purchased
  }
}

Methods of SparkMax

Now that we know how to create and initialize out SparkMax motor controller, it's time to learn how to make it move. The main way of making a motor move is by setting the power of the SparkMax via the set() method: 

            
motor.set(0.8) //0.8 is the speed percentage (80%) the motor is set to

If you find your motor spinning the wrong direction, you can either slap a big 'ol negative on the power (effective but inadvisable), or you can set the motor to be inverted (advisable) using the setInverted() method: 

            
motor.setInverted(true);

Finally, in order to stop your motor, you can either set the motor speed to zero using motor.set(0) (again, effective but inadvisable), or you can stop the motor (advisable) using the stopMotor() method: 

            
motor.stopMotor();
