Controllers

WPILib supports various controllers, including Xbox controllers, PS4 controllers, general joysticks, and even racing wheels, pedals, and gear shifts. For this tutorial, we'll just be going over Xbox and PS4 controllers. Just like for the SparkMax motor controllers, make sure you have the appropriate import.

Xbox Controllers

Imports: 

            
import edu.wpi.first.wpilibj.XboxController;

Once again, just like the SparkMax motor controllers, we have to create the XboxController object in our Robot.java class and initialize it in the Robot constructor: 

            
package frc.robot;

import com.revrobotics.spark.SparkMax;
import com.revrobotics.spark.SparkLowLevel.MotorType;

import edu.wpi.first.wpilibj.TimedRobot;
import edu.wpi.first.wpilibj.XboxController;

public class Robot extends TimedRobot {
  private SparkMax motor;
  private XboxController controller;

  public Robot() {
    motor = new SparkMax(0,MotorType.kBrushless);
    controller = new XboxController(0); //0 is the port of the driver station
                                        //that the controller is plugged into
  }
}

Methods of XboxController

The XboxController class has various functions to read button inputs on the controller. For example, each button has a getButton, getButtonPressed, and getButtonReleased method. Typically you would place these in your Robot.java teleopPeriodic() method to make a motor move or stop, as followed: 

        
  @Override
  public void teleopPeriodic() {
    if (controller.getAButton()) { //While button A is pressed on the controller,
      motor.set(0.8);              //the motor will spin at 80% speed
    }
    if (controller.getYButtonPressed()) { //When the button Y is pressed,
        motor.set(0.5);                   //the motor will spin at 50% speed
    }
    if (controller.getYButtonReleased()) { //When the button Y is released,
        motor.stopMotor();                 //the motor will stop spinning
    }
  }

The triggers and joysticks are measured as analog inputs rather than booleans. Therefore, you'd usually set them equal to some double value, like so: 

            
  @Override
  public void teleopPeriodic() {
    double leftY = controller.getLeftY(); //range of [-1,1]; back is positive
    motor.set(leftY);
    double leftTrigger = controller.getLeftTriggerAxis(); //range of [0,1]
    motor.set(leftTrigger);
  }

Methods of PS4Controller

The PS4Controller class works extremely similar to the XboxController class, with the main difference being the fact that PS4 controllers use shapes while Xbox controllers use letters. This changes the methods from getAButton(), for example, to getCrossButton()

        
  @Override
  public void teleopPeriodic() {
    if (controller.getCrossButton()) { //While button cross is pressed on the controller,
      motor.set(0.8);              //the motor will spin at 80% speed
    }
    if (controller.getSquareButtonPressed()) { //When the button square is pressed,
        motor.set(0.5);                   //the motor will spin at 50% speed
    }
    if (controller.getSquareButtonReleased()) { //When the button square is released,
        motor.stopMotor();                 //the motor will stop spinning
    }
  }

The triggers and joysticks are measured as analog inputs rather than booleans. Therefore, you'd usually set them equal to some double value, like so: 

            
  @Override
  public void teleopPeriodic() {
    double leftY = controller.getLeftY(); //range of [-1,1]; back is positive
    motor.set(leftY);
    double leftTrigger = controller.getL2Axis(); //range of [0,1]
    motor.set(leftTrigger);
  }
